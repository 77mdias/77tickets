# OPS-002 - Criar integração de persistência atômica no check-in

Data: 2026-03-29

## Objetivo
Implementar proteção atômica no consumo de ticket durante check-in para evitar inconsistência em tentativas simultâneas e manter retorno operacional previsível.

## Escopo executado
- Contrato de repositório atualizado para operação atômica:
  - `src/server/repositories/ticket.repository.contracts.ts`
  - `markAsUsedIfActive(ticketId, checkedInAt): Promise<boolean>`
- Repositório Drizzle atualizado com update condicional por status:
  - `src/server/repositories/drizzle/drizzle-ticket.repository.ts`
  - `UPDATE tickets SET status='used', checked_in_at=? WHERE id=? AND status='active'`
- Use-case de check-in ajustado para tratar corrida concorrente:
  - `src/server/application/use-cases/validate-checkin.use-case.ts`
  - quando marcação atômica falha (`false`), retorna rejeição com `reason: 'ticket_used'`
- Testes atualizados/adicionados:
  - `tests/unit/application/validate-checkin.use-case.test.ts`
  - `tests/integration/repositories/drizzle-ticket.repository.integration.test.ts`
