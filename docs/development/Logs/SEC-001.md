# SEC-001 - Cobrir RBAC completo de check-in por papel/escopo

## Objetivo
Concluir a task `SEC-001` garantindo autorização de check-in por papel (`checker`, `organizer`, `admin`, `customer`) e escopo de ownership para `organizer`, com erro estruturado e sem vazamento de detalhe interno.

## Escopo implementado
- Policy de acesso de check-in:
  - `src/server/application/security/checkin-access.policy.ts`
  - `src/server/application/security/index.ts` (export)
- Handler de check-in com gate de autorização:
  - `src/server/api/checkin/validate-checkin.handler.ts`
- Composição da rota para injeção de dependência de ownership:
  - `src/app/api/checkin/route.ts`
- Cobertura automatizada:
  - `tests/unit/server/application/security/checkin-access.policy.test.ts`
  - `tests/unit/server/api/checkin/validate-checkin.handler.test.ts`
  - `tests/integration/api/checkin/auth.test.ts`

## Regras de RBAC aplicadas
- `admin`: acesso global de check-in.
- `checker`: acesso global de check-in.
- `organizer`: acesso permitido apenas quando `actor.userId === event.organizerId`.
- `customer`: acesso negado.
