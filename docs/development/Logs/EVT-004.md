## Cenários cobertos

| # | Grupo | Teste | Intenção |
|---|-------|-------|----------|
| 1 | Cancelled event protection | Evento cancelado com lotes válidos é bloqueado | Protege a guarda `event_cancelled` em si |
| 2 | Cancelled event protection | Cancelado sem lotes → erro `event_cancelled`, não `no_lots` | Protege a **ordem** das verificações (status antes dos lotes) |
| 3 | Cancelled event protection | Cancelado sem janela de venda → `event_cancelled`, não `no_sale_window` | Cancelamento independe de configuração de lotes |
| 4 | Status transition guard | Evento já publicado é bloqueado | Protege a guarda `event_already_published` |
| 5 | Status transition guard | Matriz completa: draft→ok, published→blocked, cancelled→blocked | Living spec do state machine inteiro |
| 6 | Idempotency | Segunda chamada ao publish retorna `event_already_published` estável | Simula double-submit sem banco de dados (mock com estado) |
| 7 | Persistence fidelity | `save` é chamado com todos os campos originais + `status: "published"` | Protege o spread `{ ...event, status: "published" }` |
| 8 | Persistence fidelity | `endsAt: null` é preservado no `save` | Protege campos de data nulos contra coerção ou perda |
| 9 | Persistence fidelity | `endsAt` concreto é preservado exatamente | Complementa o teste 8 para valor não-nulo |

## Decisões de design

- **Import estático** do `createPublishEventUseCase` (não dinâmico como nos testes RED do EVT-002): a implementação já existe.
- **Arquivo autocontido**: tipos (`EventRecord`, `LotRecord`, etc.) e constantes (`EVENT_ID`, `DEFAULT_LOT`) definidos localmente, sem compartilhar com o arquivo de testes unitários. Isso garante que as regressões sobrevivam a refatorações nos unit tests.
- **Mock com estado** para o teste de idempotência (teste 6): `storedStatus` declarado no escopo do teste, `findById` lê o binding via closure. O `save` atualiza `storedStatus` para simular persistência.
- **Não foram criados arquivos de produção**: nenhuma mudança no use-case, domínio ou repositórios.
