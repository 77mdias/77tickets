## O que foi implementado

Entrega de interface mínima operacional para organizer/admin, com separação de camadas e integração apenas via endpoints existentes:

- `POST /api/events/publish`
- `POST /api/events/update-status`
- `POST /api/coupons/create`
- `POST /api/coupons/update`

Composição final:

- **Página app:** `src/app/admin/page.tsx` publica a rota `/admin`.
- **UI de gestão:** `src/features/admin/management-form.tsx` concentra os quatro fluxos operacionais (publish/update/create coupon/update coupon).
- **Client helper:** `src/features/admin/management-client.ts` monta payloads/headers (`x-actor-id`, `x-actor-role`) e trata feedback de sucesso/erro.
- **Fronteira arquitetural preservada:** UI não executa regras de ownership/publicação/cupom; backend mantém toda decisão de negócio/RBAC.
