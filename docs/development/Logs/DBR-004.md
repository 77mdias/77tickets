# DBR-004 — Testes de integração de persistência

Data: 2026-03-27
Status: ✅ Concluída

## Objetivo
Registrar a evidência do ciclo RED -> GREEN para a task `DBR-004`.

## Escopo executado
- Ampliação da cobertura de integração dos repositórios Drizzle para cenários de constraint.
- Introdução de erro tipado de persistência na camada de repositórios.
- Correção do teste de `DrizzleOrderRepository` para refletir persistência sequencial, sem alegar atomicidade inexistente.

## Evidência RED
Comando:
```bash
npx vitest run --config vitest.integration.config.ts tests/integration/repositories/drizzle-event.repository.integration.test.ts tests/integration/repositories/drizzle-order.repository.integration.test.ts tests/integration/repositories/drizzle-ticket.repository.integration.test.ts
```

Resultado:
- 3 arquivos de teste falharam.
- 3 testes falharam.
- As falhas esperadas mostraram `DrizzleQueryError` cru com códigos Postgres `23505` e `23503`.
- Motivo: os repositórios ainda não mapeavam constraints para um erro de persistência estável.

## Correção mínima aplicada
- `src/server/repositories/persistence-error.ts`
- `src/server/repositories/drizzle/map-persistence-error.ts`
- `src/server/repositories/drizzle/*.repository.ts`
- `tests/integration/repositories/*.integration.test.ts`

Mudanças:
- Criado `PersistenceError` com kinds estáveis para `unique-constraint`, `foreign-key-constraint`, `check-constraint` e `unknown`.
- Criado mapper de erros Drizzle/Neon baseado em códigos Postgres.
- Envolvidos os caminhos de escrita dos repositórios Drizzle com conversão para `PersistenceError`.
- Adicionados testes de integração para slug duplicado, código de ticket duplicado e violação de FK em itens de pedido.

## Evidência GREEN
Comandos:
```bash
npx vitest run --config vitest.integration.config.ts tests/integration/repositories/drizzle-event.repository.integration.test.ts tests/integration/repositories/drizzle-order.repository.integration.test.ts tests/integration/repositories/drizzle-ticket.repository.integration.test.ts
npm run test:integration
npm run lint
npm run build
```

Resultado:
- Suite direcionada: 3 arquivos, 21 testes, tudo passando.
- Suite completa de integração: 19 arquivos, 158 testes, tudo passando.
- `npm run lint`: sucesso.
- `npm run build`: sucesso.

## Observação
Os testes de integração dependem de conectividade com o banco Neon configurado em `TEST_DATABASE_URL`. A execução validada nesta task precisou de acesso externo ao banco de teste.
