## Decisões de design

- `organizerId` permanece como **dependência injetada** no factory (não no input), conforme padrão estabelecido em EVT-001.
- Erros lançam `AppError` (padrão do create-order); teste usa `rejects.toMatchObject({ code, details })`.
- Tipo `PublishEventUseCaseFactory` definido inline no teste para não depender do módulo inexistente.
- `lotRepository.findByEventId` incluído como dependência do factory para validar pré-condições de lotes.
- Pré-condições checadas: existência de lotes + pelo menos um lote com `saleStartsAt !== null`.
