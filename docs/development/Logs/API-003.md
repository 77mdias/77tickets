# API-003 - Testes de integração do endpoint de compra

Data: 2026-03-28

## Objetivo
Concluir a task `API-003` cobrindo o endpoint de compra de ponta a ponta na fronteira de API (`handler -> use-case -> repositories -> db`) com dados isolados por teste e erros estruturados para cenários negativos.

## Escopo executado
- Nova suíte de integração para compra:
  - `tests/integration/api/orders/create-order.endpoint.integration.test.ts`
  - Cenários cobertos:
    - sucesso (`200`) com cálculo server-side e cupom válido
    - estoque insuficiente (`409`, `details.reason = insufficient_stock`)
    - cupom inválido (`409`, `details.reason = invalid_coupon`)
    - payload inválido (`400`, erro de validação estruturado)
- Correção descoberta via TDD no adapter da API:
  - `src/server/api/create-order.handler.ts` alterado para fluxo assíncrono com `await dependencies.createOrder(...)`
  - mapeamento correto de erros assíncronos para `mapAppErrorToResponse`
- Ajuste de tipagem para refletir contrato assíncrono:
  - `src/server/application/use-cases/create-order.use-case.ts` (`CreateOrderUseCase` agora retorna `Promise<CreateOrderResult>`)
  - `tests/unit/server/api/create-order.handler.test.ts` adaptado para assertions assíncronas
