## Decisões de design

- `PublishEventInput` contém apenas `eventId` (UUID do evento a publicar).
- `organizerId` **não** faz parte do input da camada de API — segue o padrão de `checkerId` no use-case de check-in: será injetado como dependência no use-case (EVT-003), não derivado do corpo da requisição.
- Schema usa `.strict()` para rejeitar campos desconhecidos no boundary.
- `PublishEventResult` define `status: "published"` como literal — sem ambiguidade no retorno.
