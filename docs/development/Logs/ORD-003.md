## Objetivo
Concluir a task `ORD-003` implementando o use-case `createOrder` mínimo para sair de RED para GREEN, mantendo regras críticas na camada de aplicação (server-side) e sem mover lógica para handler/UI.

## Escopo executado
- Implementação do use-case em:
  - `src/server/application/use-cases/create-order.use-case.ts`
- Comportamentos implementados:
  - export de `createCreateOrderUseCase` com dependências explícitas de repositório/clock/id
  - resolução de lotes por item
  - validação de lote (`out_of_window`, `lot_not_active`, `exceeds_per_order_limit`, `insufficient_stock`)
  - cálculo de subtotal com preço do lote no servidor (ignorando preço do client)
  - validação de cupom por elegibilidade
  - aplicação de desconto (`fixed`/`percentage`) no servidor
  - persistência de pedido/itens via `orderRepository.create`
  - incremento de uso de cupom somente quando aplicado com sucesso
