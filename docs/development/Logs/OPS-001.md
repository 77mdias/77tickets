# OPS-001 - Implementar handler de check-in com mapeamento de erros

Data: 2026-03-29

## Objetivo
Concluir a task `OPS-001` expondo o fluxo de validação de check-in na camada de API com handler fino, validação de input via Zod e mapeamento de erros estruturados para respostas previsíveis no app operacional.

## Escopo executado
- Implementação do handler de check-in em:
  - `src/server/api/checkin/validate-checkin.handler.ts`
- Export do handler no barrel de API:
  - `src/server/api/index.ts`
- Testes unitários do handler:
  - `tests/unit/server/api/checkin/validate-checkin.handler.test.ts`
- Mapeamentos de erro implementados:
  - `ticket_not_found` -> `not-found` (404)
  - `unauthorized_checker` -> `authorization` (403)
  - rejeições operacionais (`event_mismatch`, `ticket_used`, `ticket_cancelled`, `ticket_expired`, `order_not_eligible`) -> `conflict` (409)
  - falha inesperada -> `internal` (500)
- Todas as respostas de rejeição incluem `details.reason` para previsibilidade operacional.
