# ADM-001 — Endpoints de publish/update para organizer/admin

**Data:** 2026-03-29  
**Status:** ✅ Concluído  
**Fase:** 005

---

## O que foi implementado

Entrega dos endpoints administrativos de evento com separação de camadas:

- `POST /api/events/publish`
- `POST /api/events/update-status`

Composição final:

- **Route adapters** validam JSON + actor headers (`x-actor-id`, `x-actor-role`).
- **Handlers** permanecem finos (parse, autorização por ownership, delegação para use-case, mapeamento de erro).
- **Use-cases** concentram regra de negócio:
  - publish segue pré-condições já existentes;
  - update de status não permite publicar via endpoint de update.

## Regras e decisões aplicadas

- Organizer só opera evento próprio; admin tem escopo global.
- `update-status` bloqueia `targetStatus = published` com conflito estável:
  - `details.reason = "publish_requires_publish_endpoint"`.
- `update-status` permite cancelamento (`draft|published -> cancelled`) e bloqueia transições inválidas.
- Erros de autorização/conflito permanecem padronizados para frontend/auditoria.
