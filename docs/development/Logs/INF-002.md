# INF-002 — Estratégia de dados para testes de integração

**Data:** 2026-03-26
**Status:** ✅ Concluída

---

## Decisão central

Testes de integração exigem `TEST_DATABASE_URL` apontando para um banco/branch Neon **separado** do banco de desenvolvimento. A variável é obrigatória; a suite falha com erro claro se não estiver configurada.

## Estratégia de isolamento

- `cleanDatabase(db)` — executa `TRUNCATE ... RESTART IDENTITY CASCADE` em todas as tabelas do schema Drizzle antes/depois de cada suite. No-op automático enquanto o schema estiver vazio (Phase 1).
- `createTestDb()` — fábrica de instância Drizzle para o banco de teste.
- `globalSetup` do Vitest — valida `TEST_DATABASE_URL` e testa conectividade uma vez antes de toda a suite, falhando rápido com mensagem acionável.

## Arquivos entregues

| Arquivo | Responsabilidade |
|---|---|
| `vitest.integration.config.ts` | Config de testes de integração com `globalSetup` |
| `tests/integration/setup/global-setup.ts` | Validação de ambiente e connectivity check |
| `tests/integration/setup/index.ts` | `createTestDb()`, `cleanDatabase()` |
| `tests/fixtures/index.ts` | Barrel de fixtures (documentado, vazio em Phase 1) |
| `tests/integration/smoke.integration.test.ts` | 3 testes de conectividade reais |
| `.env.example` | Documentação de `TEST_DATABASE_URL` |
| `package.json` | Script `test:integration` usa `vitest.integration.config.ts` |

## Setup necessário

1. Criar um branch/database de teste no [Neon Console](https://console.neon.tech).
2. Adicionar ao `.env` local (ou `.env.test`):
   ```
   TEST_DATABASE_URL=postgresql://user:pass@host/test-db?sslmode=require
   ```
3. `npm run test:integration` — deve passar com 3 testes.

## Notas técnicas

- `sql.raw()` usado em `cleanDatabase` para construir a cláusula `TRUNCATE` dinamicamente — o tagged template quotearia a lista de tabelas como parâmetro SQL, gerando SQL inválido.
- `isTable` do `drizzle-orm` filtra entradas do schema que são tabelas reais (pronto para quando Phase 2 adicionar tabelas).
- `describe.skipIf(!process.env.TEST_DATABASE_URL)` no smoke test atua como guarda secundário quando o arquivo é executado fora do config de integração.
