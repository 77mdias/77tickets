# UX-001 - Formulario minimo de checkout integrado ao endpoint

Data: 2026-03-28

## Objetivo
Conectar a UI minima de checkout ao endpoint de criacao de pedido, mantendo regras de negocio no backend e exibindo erro estruturado de forma compreensivel no frontend.

## Escopo executado
- Endpoint HTTP criado em `src/app/api/orders/route.ts` (`POST /api/orders`).
- Adapter de rota criado em `src/server/api/orders/create-order.route-adapter.ts` para:
  - parse de JSON no boundary;
  - injecao de `customerId` server-side para modo demo;
  - propagacao de erro estruturado (`validation`, `conflict`, etc.).
- Feature de checkout criada:
  - `src/features/checkout/checkout-form.tsx`
  - `src/features/checkout/checkout-client.ts`
- Home atualizada para usar o formulario:
  - `src/app/page.tsx`
- Guardrail de lint ajustado para permitir imports de server em `src/app/api/**`:
  - `eslint.config.mjs`
  - `tests/unit/architecture/eslint-guardrails.test.ts`

## Decisao de seguranca para o demo
- Fonte de identidade: `DEMO_CUSTOMER_ID` no servidor (com fallback UUID estavel).
- O client nao define identidade efetiva: se enviar `customerId`, o valor e sobrescrito no boundary de rota.
- Motivacao: evitar confiar em identidade informada no navegador, mantendo compatibilidade com API-004.
