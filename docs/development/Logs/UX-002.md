# UX-002 - Instrumentar logs e métricas básicas de checkout

Data: 2026-03-28

## Objetivo
Instrumentar observabilidade mínima do checkout para medir sucesso/falha, latência e categorias de erro, preservando respostas seguras e sem vazamento de dados sensíveis.

## Escopo executado
- Handler de checkout com telemetria estruturada:
  - `src/server/api/create-order.handler.ts`
  - evento `checkout.create_order` com `status`, `latencyMs`, `errorCode`, `outcome` e contexto seguro
- Use-case com hook opcional de observabilidade:
  - `src/server/application/use-cases/create-order.use-case.ts`
  - evento `checkout.create_order.use_case` para sucesso/falha com categorização de erro
- Adapter de observabilidade na infraestrutura:
  - `src/server/infrastructure/observability/checkout-observability.ts`
  - export em `src/server/infrastructure/observability/index.ts` e `src/server/infrastructure/index.ts`
- Injeção de dependência no endpoint:
  - `src/app/api/orders/route.ts`
- Runbook operacional:
  - `docs/infrastructure/checkout-observability.md`
